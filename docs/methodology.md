# 방법론 노트

## 0. 기준 문서

- `AGENTS.md`: 작업 기준과 품질 기준
- `SOURCE.md`: 참고 경로와 컨텐츠 재구성 기준

## 1. 프로젝트 목적

확률분포 가정이 금융 변수의 미래 범위와 손실확률에 미치는 영향을 시뮬레이션한다.

## 2. 핵심 질문

평균과 변동성 가정이 바뀔 때 1년 뒤 포트폴리오 가치 분포는 어떻게 달라지는가?

## 3. 입력 데이터

- 파일: `data/sample/simulation_assumptions.csv`
- 성격: 구조 검증용 합성 샘플
- 목적: 계산 흐름과 결과 해석을 작은 표본으로 먼저 검증

## 4. 계산 절차

1. 로그정규 terminal value를 난수 시드와 함께 생성해 재현성을 확보한다.
2. 평균, 분위수, 손실확률을 비교해 분포 가정의 영향을 직관적으로 설명한다.
3. 입력값은 분포 가정의 차이를 비교하도록 구성한 합성 시나리오다.

## 5. 복습 자료 생성 절차

복습 자료는 `data/sample/review_scenarios.csv`의 합성 시나리오를 사용합니다. baseline 입력과 별도 파일로 둔 이유는 연구 결과표와 학습용 민감도 표를 구분해, 어느 숫자가 검증용 baseline이고 어느 숫자가 복습용 연습 자료인지 혼동하지 않기 위해서입니다.

```bash
python -m src.generate_review_materials
```

위 명령은 같은 GBM terminal value 공식을 사용하되, 다음 네 가지 산출물을 만듭니다.

- `review_distribution_summary.csv`: 평균, 5%/25%/50%/75%/95% 분위수, IQR, 손실확률, 20% 이상 상승확률
- `review_convergence_check.csv`: 경로 수 250, 500, 1000, 2000, 최종 경로 수에서 평균 표준오차와 신뢰구간 반폭
- `review_flashcards.csv`: seed, 경로 수, 분위수, 손실확률, 룩어헤드 바이어스 등 핵심 개념 질문/답
- `review_practice_questions.csv`: 결과표를 근거로 직접 설명해 보는 자가 점검 문제와 답안

경로 수별 수렴성 표는 동일 seed에서 앞부분 난수열이 유지되도록 생성합니다. 이렇게 하면 경로 수 증가에 따른 표본오차 감소를 비교할 때 난수 조건이 불필요하게 바뀌는 문제를 줄일 수 있습니다.

## 6. 산출물 해석

`outputs/tables/baseline_results.csv`는 프로젝트별 baseline 결과입니다. 이 표는 최종 연구 결론이 아니라, 이후 실제 데이터 연결과 상세 분석을 시작하기 위한 구조 검증용 산출물입니다.

복습용 `review_*.csv` 파일도 같은 성격의 합성 자료입니다. 개념 복습과 재현성 확인에는 사용할 수 있지만, 실제 투자 성과나 포트폴리오 적합성을 주장하는 근거로 사용하면 안 됩니다.

## 7. 한계

- 합성 데이터 기반이므로 통계적 유의성이나 투자 가능성을 주장하지 않습니다.
- 거래비용, 세금, 슬리피지, 유동성 제약은 일부 프로젝트에서만 단순 반영했습니다.
- 프로젝트별 정교한 모델링은 별도 후속 작업에서 확장해야 합니다.
